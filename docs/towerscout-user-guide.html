User Guide

A practical guide to choosing a provider, defining a search area, running detection, reviewing likely towers, and saving results.

Provider Search Area Detection Review Manual Towers Export And Restore Help

Choose A Provider

TowerScout supports Google Maps and Azure Maps. Only one valid provider key is required to start. If both are configured, use the provider selector to choose the imagery source for the current workflow.

Browser map SDK keys are visible to someone who can access the running app. V1 RC1 assumes site/user-owned restricted keys. Do not share provider keys, .env, browser network traces, screenshots that reveal keys, cached provider responses, or raw logs unless your site has approved handling for that material.

Search Or Define An Area

You can move the map manually or search by street address, city, neighborhood, or ZIP code. ZIP code search requires imported ZIP-code asset data.

Circle Search

  1. Search for or navigate to the location.
  2. Enter a radius in meters.
  3. Select Circle.
  4. Select Estimate tiles.

Custom Search Area

Custom search areas are polygons used to tell TowerScout where to run detection. They are different from manual tower polygons.

  • Azure Maps: double-click to complete the polygon.
  • Google Maps: right-click outside to complete the polygon.

Estimate And Run Detection

Select Estimate tiles before Find towers. The estimate tells you how many imagery tiles TowerScout expects to process and the rough expected processing time.

During detection, TowerScout downloads imagery, runs the detector, applies secondary classification where configured, removes duplicates, and geocodes results. A progress overlay shows the current phase and detail. You can cancel an active run if needed.

Review Results

Results appear on the map and in the detection list. Use the list to center or highlight a detection, uncheck likely false positives before export, adjust the minimum confidence slider, and step through detections or tiles with review controls.

The confidence slider controls what is visible and exported. A higher threshold shows fewer detections. A lower threshold shows more detections and may include more false positives.

Add Manual Tower Detections

Manual tower detections are corrections you add after a detection run. They are not the same as custom search-area polygons.

  1. Run detection first.
  2. Select Add Towers.
  3. Draw around the tower.
  4. Complete the polygon: Azure Maps uses double-click; Google Maps uses right-click outside.
  5. Select Save Towers.

Manual towers are included in CSV, KML, and dataset exports when saved.

Export And Restore

CSV And KML

Use Download results to download detections.csv and detections.kml.

Dataset ZIP

Use Download dataset to save the current tiles, labels, metadata, and manual additions as dataset.zip. Dataset ZIPs may contain sensitive locations, investigation context, imagery tiles, and manual corrections.

Restore

Use Restore dataset to reload a previously exported dataset.zip into the current session. After restore, review the provider selection, map framing, detections, and manual towers before exporting again.

Stop, Resume, And Get Help

Stop TowerScout with .\scripts\stop.cmd. Restart later with .\start.bat.

If support asked you to run TowerScout with a specific engine, use the same -Engine value on start, stop, status, logs, and asset-import commands because Docker and Podman use separate named volumes.

When reporting a problem, include what you were trying to do, the release package version, whether you used Docker or Podman, the readiness state from scripts\status.cmd, and a reviewed/redacted log summary if support asks for it.

Do not share provider keys, .env, raw logs, raw screenshots, browser network traces, cached provider responses, uploaded investigation files, exported datasets, named-volume contents, or sensitive AOIs unless your site has an approved support-handling procedure.
