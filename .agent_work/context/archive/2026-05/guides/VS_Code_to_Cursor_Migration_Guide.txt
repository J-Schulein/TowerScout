CTowerScout Development Environment Migration: VS Code to Cursor
================================================================

This guide provides step-by-step instructions for migrating your TowerScout development 
environment from Visual Studio Code to Cursor while preserving all project structure, 
configurations, and development workflow.

OVERVIEW
--------
Cursor is a VS Code fork designed specifically for AI-assisted development. It maintains 
compatibility with VS Code extensions and configurations while adding AI coding features.
Your TowerScout project structure will be preserved completely.

PREREQUISITES
-------------
- Cursor installed on your system (download from cursor.sh)
- Current VS Code workspace functional and backed up
- Git repository in clean state
- Python virtual environment activated

MIGRATION STEPS
===============

STEP 1: BACKUP CURRENT ENVIRONMENT
----------------------------------
1. Create a backup of your entire TowerScout directory:
   - Copy C:\Users\bg90\TowerScout to C:\Users\bg90\TowerScout_backup

2. Export VS Code settings and extensions:
   - Open VS Code Command Palette (Ctrl+Shift+P)
   - Run "Settings Sync: Show Settings"
   - Note your current settings sync status
   - Export settings manually if needed:
     * File > Preferences > Settings (gear icon) > Export Settings

3. Document current extensions:
   - Open Extensions view (Ctrl+Shift+X)
   - Create list of installed extensions (especially TowerScout-specific ones)

4. Backup workspace configuration:
   - Copy .vscode/ directory contents (if exists)
   - Note any workspace-specific launch configurations

STEP 2: INSTALL AND CONFIGURE CURSOR
------------------------------------
1. Download Cursor from cursor.sh and install

2. Initial Cursor setup:
   - Launch Cursor for first time
   - Sign in with GitHub/Google account for AI features
   - Configure AI model preferences (Claude, GPT-4, etc.)

3. Import VS Code settings (if using Settings Sync):
   - Enable Settings Sync in Cursor
   - Sign in with same account used in VS Code
   - Allow settings/extensions to sync

4. Manual settings migration (if not using sync):
   - Copy settings.json from VS Code user directory to Cursor
   - VS Code location: %APPDATA%\Code\User\settings.json
   - Cursor location: %APPDATA%\Cursor\User\settings.json

STEP 3: PROJECT STRUCTURE PRESERVATION
--------------------------------------
1. Close VS Code completely

2. Open TowerScout project in Cursor:
   - Launch Cursor
   - File > Open Folder
   - Select C:\Users\bg90\TowerScout

3. Verify project structure integrity:
   - Confirm all directories visible (.agent_work, webapp, tests, etc.)
   - Check that hidden files are shown (.gitignore, .env, etc.)
   - Verify flask_session and cache directories preserved

4. Test file associations:
   - Ensure .py files open with Python syntax highlighting
   - Verify .md files render with markdown preview
   - Check .ipynb notebooks open correctly

STEP 4: DEVELOPMENT ENVIRONMENT SETUP
-------------------------------------
1. Configure Python interpreter:
   - Open Command Palette (Ctrl+Shift+P)
   - "Python: Select Interpreter"
   - Choose your TowerScout virtual environment:
     C:\Users\bg90\TowerScout\.venv\Scripts\python.exe

2. Verify terminal integration:
   - Open integrated terminal (Ctrl+`)
   - Confirm it activates virtual environment automatically
   - Test: python --version should show correct Python version

3. Configure workspace settings:
   - Create/update .vscode/settings.json in project root:
   {
     "python.pythonPath": ".venv/Scripts/python.exe",
     "python.terminal.activateEnvironment": true,
     "python.linting.enabled": true,
     "python.linting.flake8Enabled": true,
     "python.formatting.provider": "black",
     "python.testing.pytestEnabled": true,
     "python.testing.pytestPath": "pytest"
   }

2. Web development extensions:
   - HTML CSS Support

3. TowerScout-specific extensions:

   - GitLens - Enhanced git integration

   - Docker (if using containerization)

4. AI coding enhancements (Cursor-specific):

   - Set up custom AI instructions for TowerScout context

TROUBLESHOOTING
===============

Common Issues and Solutions:

1. Python interpreter not found:
   - Manually select interpreter: Ctrl+Shift+P > "Python: Select Interpreter"
   - Verify virtual environment is in correct location

2. Extensions not working:
   - Reinstall Python extensions one by one
   - Check extension compatibility with Cursor
   - Restart Cursor after installing extensions

3. Terminal not activating virtual environment:
   - Check terminal.integrated.env.windows settings
   - Manually activate: source .venv/Scripts/activate (in Git Bash)
   - Verify python.terminal.activateEnvironment is true

4. AI features not working:
   - Check internet connection
   - Verify Cursor account is properly signed in
   - Check AI model availability and rate limits

5. Git integration issues:
   - Verify git is in system PATH
   - Check git credentials are properly configured
   - Test git operations from external terminal first

ROLLBACK PLAN
=============

If migration causes issues:

1. Immediate rollback:
   - Close Cursor
   - Restore TowerScout_backup directory
   - Reopen project in VS Code

2. Partial rollback:
   - Keep project in Cursor
   - Use VS Code for specific tasks if needed
   - Gradually transition over time

3. Hybrid approach:
   - Use both editors for different purposes
   - Cursor for AI-assisted development
   - VS Code for complex debugging or familiar workflows
