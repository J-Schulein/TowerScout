# User Testing Artifacts

Store raw tester-provided evidence here.

## Rules

- Use one dated folder per issue report or rerun.
- Keep original screenshots, logs, and recordings unchanged.
- Add a short note only when the artifact itself needs context.
- Never store API keys, full `.env` files, or other secrets here.

## Recommended Contents

- screenshot of the failing step
- PowerShell or terminal output saved as `.txt`
- browser console or network export when relevant
- short note saying what step failed and when

## Naming

Use:

`YYYY-MM-DD-ut-###-short-slug/`
