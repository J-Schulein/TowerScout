TOWERSCOUT DEVELOPMENT ENVIRONMENT SETUP GUIDE
==============================================

This guide provides step-by-step instructions to set up a complete TowerScout development environment on a new system and continue work from the current state.

SYSTEM REQUIREMENTS
==================

Hardware Requirements:
- 8GB+ RAM (16GB recommended for GPU processing)
- 10GB+ available disk space
- GPU with CUDA support (optional - CPU fallback available)
- Reliable internet connection for downloading dependencies and model weights

Software Requirements:
- Windows 10/11, macOS 10.15+, or Linux (Ubuntu 20.04+ recommended)
- Git 2.30+
- Python 3.12+ (3.12.x specifically recommended)
- VS Code (recommended IDE) or PyCharm
- Docker Desktop (optional but recommended for deployment testing)

STEP 1: INITIAL SYSTEM SETUP
============================

1.1 Install Git
---------------
Windows: Download from https://git-scm.com/download/win
macOS: Install via Homebrew: brew install git
Linux: sudo apt update && sudo apt install git

Verify installation: git --version

1.2 Install Python 3.12
-----------------------
Windows: 
- Download from https://www.python.org/downloads/
- During installation, check "Add Python to PATH"
- Verify: python --version and pip --version

macOS:
- Use Homebrew: brew install python@3.12
- Add to PATH if needed
- Verify: python3.12 --version

Linux (Ubuntu):
- sudo apt update
- sudo apt install software-properties-common
- sudo add-apt-repository ppa:deadsnakes/ppa
- sudo apt install python3.12 python3.12-pip python3.12-venv
- Verify: python3.12 --version

1.3 Install VS Code (Recommended)
---------------------------------
Download from: https://code.visualstudio.com/
Install recommended extensions:
- Python
- Pylance
- Python Debugger
- GitLens
- Docker (if using Docker)

1.4 Install Docker Desktop (Optional)
------------------------------------
Download from: https://www.docker.com/products/docker-desktop
Required for testing containerized deployment
Follow platform-specific installation instructions

STEP 2: CLONE REPOSITORY AND SETUP PROJECT
==========================================

2.1 Clone the Repository
------------------------
Open terminal/command prompt and navigate to desired project directory:

git clone https://github.com/J-Schulein/TowerScout.git
cd TowerScout

# Switch to the improvements branch (where current work is)
git checkout improvements

# Verify you're on the correct branch
git branch
# Should show: * improvements

2.2 Set Up Python Virtual Environment
------------------------------------
# Create virtual environment
python -m venv venv

# Activate virtual environment
# Windows (Command Prompt):
venv\Scripts\activate

# Windows (PowerShell):
venv\Scripts\Activate.ps1

# macOS/Linux:
source venv/bin/activate

# Verify activation - prompt should show (venv)

2.3 Install Python Dependencies
------------------------------
# Upgrade pip first
pip install --upgrade pip

# Install main dependencies
cd webapp
pip install -r requirements.txt

# Install development dependencies (from project root)
cd ..
pip install -r requirements-dev.txt

# If requirements-dev.txt doesn't exist yet, install these manually:
pip install pytest pytest-cov black flake8 mypy

STEP 4: ENVIRONMENT CONFIGURATION
=================================

4.1 Create Environment Variables File
------------------------------------
Create .env file by copying the example template:

# Copy the template (recommended)
cp webapp/.env.example webapp/.env  # Linux/macOS
copy webapp\.env.example webapp\.env  # Windows

# OR manually create .env file in webapp/ directory
# touch webapp/.env  # Linux/macOS

Edit webapp/.env with your actual values:

# Required API Keys (get from cloud providers)
GOOGLE_API_KEY=your_google_maps_api_key_here
AZURE_MAPS_SUBSCRIPTION_KEY=your_azure_maps_subscription_key_here
BING_API_KEY=your_bing_maps_api_key_here  # Legacy provider

# Required Flask Configuration
FLASK_SECRET_KEY=your_flask_secret_key_here  # Generate with: python -c "import secrets; print(secrets.token_hex(32))"
FLASK_ENV=development
FLASK_DEBUG=true

# Application Settings (optional, defaults provided)
MAX_TILES=100000
MAX_TILES_SESSION=100000
UPLOAD_FOLDER=uploads

4.2 Obtain API Keys
------------------
Google Maps API:
1. Go to https://console.cloud.google.com/
Azure Maps API (Primary - OPERATIONAL):
1. Go to https://portal.azure.com/
2. Create Azure Maps Account resource
3. Navigate to Keys and Endpoint
4. Copy Primary Key
5. Add key to .env file as AZURE_MAPS_SUBSCRIPTION_KEY
6. ✅ Azure Maps provider is fully operational with 32% performance improvement

Google Maps API (Alternative):
1. Go to https://console.cloud.google.com/
2. Create new project or select existing
3. Enable Maps JavaScript API and Static Maps API
4. Create API key in Credentials section
5. Add key to .env file as GOOGLE_API_KEY

Bing Maps API (DEPRECATED):
1. ⚠️  Bing Maps provider is deprecated and no longer recommended
2. Azure Maps provides equivalent functionality with better performance
3. Legacy support maintained for existing configurations only
4. Migrate to Azure Maps for new deployments

4.3 Generate Secret Key
----------------------
# Generate secure secret key for Flask sessions
python -c "import secrets; print(secrets.token_hex(32))"
# Copy output to SECRET_KEY in .env file

STEP 9: TROUBLESHOOTING COMMON ISSUES
=====================================

9.1 Python/Pip Issues
---------------------
Issue: "python not found"
Solution: Ensure Python is in PATH, use full path, or try python3/python3.12

Issue: "pip install fails"
Solution: Upgrade pip: python -m pip install --upgrade pip

Issue: "Permission denied"
Solution: Use virtual environment, avoid sudo/admin installation

9.2 Model Loading Issues
-----------------------
Issue: "Model file not found"
Solution: Verify model files exist in correct directories with correct names

Issue: "CUDA out of memory"
Solution: Reduce batch size in ts_yolov5.py or force CPU mode

Issue: "Torch/CUDA version mismatch"
Solution: Reinstall PyTorch with correct CUDA version for your GPU

9.3 API Key Issues
-----------------
Issue: "API key not loaded"
Solution: Verify webapp/.env file exists and contains correct keys

Issue: "Maps not loading"
Solution: Check API key validity and quota limits in cloud console

Issue: "Azure Maps authentication failed"
Solution: Verify AZURE_MAPS_SUBSCRIPTION_KEY in .env file, check Azure portal for key status

Issue: "No map providers available"
Solution: At least one API key (Google, Azure, or Bing) must be configured

9.4 Flask Application Issues
---------------------------
Issue: "Port already in use"
Solution: Kill process on port 5000 or use different port

Issue: "Template not found"
Solution: Ensure working directory is webapp/ when running towerscout.py

Issue: "Session errors"
Solution: Clear flask_session directory and restart application

STEP 10: DEVELOPMENT WORKFLOW
============================

10.1 Daily Development Routine
------------------------------
1. Pull latest changes: git pull origin improvements
2. Activate virtual environment: venv/Scripts/activate (Windows) or source venv/bin/activate (Unix)
3. Check for dependency updates: pip install -r requirements.txt
4. Run tests to ensure everything works: pytest tests/
5. Start development server: cd webapp && python towerscout.py dev

10.2 Making Changes
------------------
1. Create feature branch: git checkout -b feature/your-feature-name
2. Make changes following project coding standards
3. Run tests: pytest tests/
4. Run linting: flake8 webapp/
5. Format code: black webapp/
6. Commit changes: git add . && git commit -m "descriptive message"
7. Push to repository: git push origin feature/your-feature-name

10.3 Testing Workflow

EMERGENCY CONTACTS AND BACKUP PLANS
===================================

If you encounter issues:
1. Check this setup guide troubleshooting section
2. Review error messages carefully and search online
3. Check GitHub Issues for similar problems
4. Create new GitHub Issue with detailed error information
5. Refer to original TowerScout documentation in repository

Backup Development Options:
- Use Docker container for isolated environment
- Try different Python version if 3.12 unavailable
- Use cloud development environment (GitHub Codespaces, etc.)
- Set up on different operating system if needed

Data Backup:
- All code is in Git repository (ensure regular commits)
- Model weights can be re-downloaded if lost
- Configuration in .env file should be backed up separately
- No critical data stored locally (stateless application)

This guide should get you fully set up and ready to continue TowerScout development from exactly where you left off. The key is ensuring all dependencies are installed correctly and the environment variables are configured properly.
