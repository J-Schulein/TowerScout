TOWERSCOUT PROJECT OVERVIEW AND IMPLEMENTATION PLAN
==================================================

EXECUTIVE SUMMARY
================

TowerScout is a Flask web application for detecting cooling towers from satellite imagery using YOLOv5 and EfficientNet machine learning models. Originally developed as a graduate school project, it has been successfully used in 12+ Legionnaires' disease outbreak investigations across 8 states. The current initiative focuses on improving the application for public accessibility while maintaining its proven detection capabilities.

PROJECT BACKGROUND
==================

Current State: Student prototype requiring technical setup (manual downloads, CLI commands, API key files)
Target State: User-friendly, publicly accessible tool with Docker containerization and multi-provider API support
Timeline: 10-14 weeks across 5 development phases
Progress: 6 of 26 tasks completed (23% complete)

**PROGRESS UPDATE - Azure Maps Migration**: Azure Maps provider implementation is complete with validated 32% performance improvement and 0.1-meter coordinate accuracy. The coordinate system transformation challenge (lng,lat vs lat,lng) has been successfully solved. Current focus is Azure Key Vault integration for enterprise deployment support.

ARCHITECTURE OVERVIEW
====================

Core Components:
- webapp/towerscout.py - Main Flask application with session-based processing
- webapp/ts_yolov5.py - YOLOv5 detector wrapper (batch_size=16, lazy-loaded)
- webapp/ts_en.py - EfficientNet secondary classifier (threshold=0.5, lazy-loaded)
- webapp/ts_maps.py - Abstract map provider interface
- webapp/ts_gmaps.py & webapp/ts_bmaps.py - Google/Bing Maps implementations
- webapp/ts_events.py - Event handling and progress tracking system
- webapp/ts_imgutil.py - Image processing utilities and coordinate transforms
- webapp/ts_zipcode.py - Geographic boundary validation using Census TIGER data
- Model/ - Jupyter notebooks for training and evaluation
- SyntheticData/ - Dataset augmentation tools
- TowerScoutSite/ - Static marketing site

Data Flow Process:
1. User draws polygon on map → validated against zipcode boundaries
2. Backend tiles area into 640x640px chunks via ts_imgutil.make_tiles()
3. Tile intersection check with tileIntersectsPolygons()
4. Downloads satellite imagery asynchronously via map APIs
5. YOLOv5 detection in GPU batches (16 images/batch, semaphore-controlled)
6. EfficientNet classifier filters results (threshold=0.5)
7. Coordinate transformation back to lat/lng with confidence scores

CRITICAL SECURITY ISSUES
========================

IMMEDIATE ATTENTION REQUIRED:

1. API Key Vulnerability (RESOLVED)
   - Problem: API keys stored in plain text (apikey.txt) committed to repository
   - Impact: Exposed Google/Bing API keys, potential billing fraud
   - Solution: Migrated to environment variables with .env configuration
   - Status: COMPLETED (November 30, 2025)

2. Input Validation (RESOLVED)
   - Problem: No validation on polygon coordinates, file uploads, API parameters
   - Impact: Potential injection attacks, system crashes, data corruption
   - Solution: Comprehensive ts_validation.py module with sanitization
   - Status: COMPLETED (December 2, 2025)

3. Error Handling (IN PROGRESS)
   - Problem: No try/catch blocks around API calls or model inference
   - Impact: Unhandled exceptions crash application, poor user experience
   - Solution: Structured error handling with ts_errors.py and ts_logging.py
   - Status: NEXT PRIORITY

AZURE MAPS MIGRATION REQUIREMENTS:

**Coordinate System Challenge**: Azure Maps uses lng,lat order (GeoJSON standard) while existing Google/Bing Maps use lat,lng order. This requires:
- Coordinate transformation in ts_azure_maps.py provider implementation
- Comprehensive validation to ensure ML detection accuracy preserved
- Testing with known cooling tower locations across all providers
- Side-by-side provider comparison for detection consistency

**Authentication Architecture**: Dual authentication system supporting:
- Standard Azure Maps subscription keys (development/simple deployments)
- Azure Key Vault integration (enterprise deployments with Managed Identity)
- Fallback chain: Key Vault → Environment Variables → Error
- Container deployment with Azure SDK and DefaultAzureCredential

**API Structure Changes**:
- Bing Maps (path-based): /imagery/map/aerial/{z}/{x}/{y}
- Azure Maps (query-based): ?center={lng},{lat}&zoom={z}
- Remove metadata dependencies (vintage date features)
- Implement provider fallback system (Azure → Google → Legacy Bing)

TECHNICAL SPECIFICATIONS
=======================
